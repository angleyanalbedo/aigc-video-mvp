# 电商 AIGC 带货视频系统 - 部署文档

## 1. 项目概述

本项目是一个电商 AIGC 带货视频生成系统，包含前端和后端两个部分。

- **前端**：React + Vite + TypeScript + Ant Design
- **后端**：Node.js + Express

## 2. 本地 Docker 部署

### 2.1 前置条件

确保你的系统已安装：
- Docker
- Docker Compose

### 2.2 快速启动

1. 克隆项目代码

2. 配置环境变量（可选，用于真实 API）：
   ```bash
   cp .env.example .env
   # 编辑 .env 文件，填入你的火山引擎 API 密钥
   ```

3. 使用 Docker Compose 启动服务：
   ```bash
   docker-compose up -d --build
   ```

4. 访问应用：
   - 前端界面：http://localhost
   - 后端 API：http://localhost:3001
   - 健康检查：http://localhost:3001/api/health

### 2.3 常用命令

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v

# 重启服务
docker-compose restart
```

## 3. GitHub Actions CI/CD

### 3.1 配置 Secrets

在 GitHub 仓库的 Settings → Secrets and variables → Actions 中添加以下 Secrets：

| Secret 名称 | 说明 |
|------------|------|
| `DOCKER_HUB_USERNAME` | Docker Hub 用户名 |
| `DOCKER_HUB_TOKEN` | Docker Hub 访问令牌 |

### 3.2 工作流说明

CI/CD 工作流会在以下情况触发：
- 推送到 `main` 或 `master` 分支
- 向 `main` 或 `master` 分支发起 Pull Request

工作流执行以下步骤：
1. 检出代码
2. 设置 Node.js 环境
3. 安装依赖并构建前端
4. 安装后端依赖
5. 构建 Docker 镜像
6. （可选）推送到 Docker Hub

## 4. 生产环境部署

### 4.1 使用 Docker Compose

1. 在服务器上克隆代码
2. 配置环境变量
3. 启动服务：
   ```bash
   docker-compose -f docker-compose.yml up -d
   ```

### 4.2 使用拉取的镜像

如果你配置了 Docker Hub 推送，可以直接使用：

```yaml
version: '3.8'
services:
  server:
    image: your-docker-username/aigc-video-server:latest
    ports:
      - "3001:3001"
    # ... 其他配置

  frontend:
    image: your-docker-username/aigc-video-frontend:latest
    ports:
      - "80:80"
    # ... 其他配置
```

## 5. 健康检查

系统内置了健康检查功能：

- **后端健康检查**：`GET /api/health`
- **前端健康检查**：访问首页

Docker Compose 会自动监控服务健康状态。

## 6. 数据持久化

Docker Compose 配置了以下数据卷用于数据持久化：
- `server_uploads`：上传的素材文件
- `server_outputs`：生成的视频文件
- `server_temp`：临时文件

## 7. 故障排查

### 7.1 服务无法启动

1. 检查端口是否被占用：
   ```bash
   netstat -tulpn | grep -E ':(80|3001)'
   ```

2. 查看日志：
   ```bash
   docker-compose logs server
   docker-compose logs frontend
   ```

### 7.2 前端无法连接后端

确保：
1. 后端服务已健康启动
2. 前端的 nginx 配置正确
3. 网络连接正常

### 7.3 视频生成失败

检查：
1. 环境变量是否正确配置
2. API 密钥是否有效
3. 网络连接是否正常
