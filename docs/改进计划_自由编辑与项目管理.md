# 改进计划：自由编辑工作台 + 项目管理 + SQLite 持久化

> 创建时间：2026-05-22
> 状态：实施中

---

## 一、背景与目标

### 当前问题

| 问题 | 说明 |
|------|------|
| 步骤引导式流程 | 用户必须按 素材→剧本→创作→预览 四步走，无法自由跳转 |
| 无项目管理 | 无法保存/加载/管理多个视频项目，每次都要从头开始 |
| 内存存储 | 任务、素材等数据存在内存 Map 中，服务重启即丢失 |
| 缺乏自由度 | 无法同时查看素材、剧本、分镜，操作效率低 |

### 目标

1. **自由编辑工作台** — 所有功能在同一页面，用户自由操作
2. **项目管理** — 创建/保存/加载/删除/切换项目
3. **SQLite 持久化** — 所有业务数据持久化存储

### 2.2 面板详细设计

| 面板 | 功能 | 操作 |
|------|------|------|
| **素材面板** | 素材管理 | 上传、从素材库选择、标签筛选、Embedding 检索 |
| **剧本面板** | 剧本编辑 | AI 生成、手动编辑、分镜增删、台词改写 |
| **分镜面板** | 分镜编辑 | 时间轴、拖拽排序、时长调整、素材替换 |
| **预览/生成面板** | 视频生成 | 参数配置、一键生成、实时预览、导出 |

## 三、项目管理设计

### 3.1 项目数据模型

- projects: id(TEXT PK), name(TEXT), description(TEXT), status(TEXT DEFAULT 'draft'), product_info(TEXT JSON), script(TEXT JSON), settings(TEXT JSON), created_at, updated_at

### 3.2 项目管理功能

| 功能 | API | 说明 |
|------|-----|------|
| 创建项目 | POST /api/projects | 新建空白项目 |
| 项目列表 | GET /api/projects | 获取所有项目 |
| 项目详情 | GET /api/projects/:id | 获取项目完整数据 |
| 更新项目 | PUT /api/projects/:id | 保存项目修改 |
| 删除项目 | DELETE /api/projects/:id | 删除项目 |
