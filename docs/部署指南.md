# AIGC 视频生成系统 - 部署指南

## 快速部署（推荐）

### 使用 Docker Compose 部署

```bash
cd /workspace
chmod +x deploy.sh
./deploy.sh
```

部署完成后访问：
- 🌐 前端地址: http://localhost
- 🔧 后端地址: http://localhost:3001
- 📊 健康检查: http://localhost:3001/api/health

### 常用 Docker 命令

```bash
# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 重新构建并启动
docker-compose up -d --build
```

## 本地开发部署

### 前置条件
- Node.js >= 18.0.0
- npm >= 9.0.0

### 步骤 1: 启动后端服务

```bash
cd /workspace/server

# 安装依赖
npm install

# 配置环境变量（复制 .env.example 为 .env 并填入 API 密钥）
cp .env.example .env

# 启动服务
npm start
```

后端服务将运行在 http://localhost:3001

### 步骤 2: 启动前端服务

```bash
cd /workspace/frontend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env

# 启动开发服务器
npm run dev
```

前端服务将运行在 http://localhost:5173（Vite 默认）

### 步骤 3: 访问应用

打开浏览器访问 http://localhost:5173

## 环境变量配置

### 后端 (.env)

## 故障排查

### 问题 1: 前端无法连接后端 API

**症状**: 浏览console显示 "Failed to fetch"

**解决方案**:
1. 确认后端服务正在运行: `curl http://localhost:3001/api/health`
2. 如果使用 Docker，查看日志: `docker-compose logs server`
3. 检查网络连接: 确保前端和后端在同一个网络中

### 问题 2: 视频生成失败

**症状**: API 调用返回错误

**解决方案**:
1. 检查火山方舟 API 密钥是否正确配置
2. 确认 API 额度充足
3. 查看后端日志获取详细错误信息

### 问题 3: 端口被占用

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :3001  # 后端端口
lsof -i :5173  # 前端端口

# 终止占用进程
kill -9 <PID>
```
