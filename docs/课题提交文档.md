# 后端运行在 http://localhost:3001

### 关键工程难点与解决方案

> 至少 2–3 个，如并发、延迟、数据清洗、上下文管理、前后端联调、部署问题等。

#### 难点 1：多 Agent 编排与任务状态管理

- **问题**：视频生成流程涉及脚本生成 → 分镜规划 → 素材匹配 → 视频合成 → 合规检测等多个步骤，各步骤可能异步执行，状态管理复杂。
- **解决方案**：设计了基于 Master Agent + 子 Agent 的编排架构，通过数据库任务表统一追踪每个任务的状态（pending / running / completed / failed），支持断点续传和失败重试。

#### 难点 2：大模型调用延迟与容错

- **问题**：LLM 调用和视频生成 API 耗时较长（视频生成可达分钟级），且可能出现超时或服务不可用。
- **解决方案**：引入重试机制（指数退避）、异步任务队列、前端轮询进度反馈，以及模型调用的超时兜底策略。视频生成支持替换为轻量级模型进行快速验证。

#### 难点 3：前后端联调与实时进度反馈

- **问题**：视频生成过程耗时较长，用户需要实时感知任务进度，但 HTTP 轮询效率低。
- **解决方案**：实现 WebSocket 实时推送任务进度，前端通过进度条和状态卡片直观展示当前执行阶段（脚本生成中 → 素材匹配中 → 视频合成中 → 合规检测中）。

### 项目亮点 / 创新点

> 建议 3 条以内，突出与同类方案相比的差异化。

1. **多 Agent 协作架构**：采用 Master Agent 统一编排 + 专业子 Agent 分工的架构，各 Agent 可独立迭代和替换，具备良好的可扩展性。
2. **数据驱动的创作闭环**：从爆款视频分析 → 模板提炼 → 策略生成 → 效果归因，形成完整的数据驱动创作优化闭环，而非一次性生成工具。
3. **Copilot 自然语言创作**：通过无限画布与自然语言交互降低视频创作门槛，用户无需专业知识即可通过描述需求完成视频制作。

## 七、FAQ

| 问题 | 回答 |
|------|------|
| **技术栈必须严格遵守吗？** | 核心技术栈（React、Node.js、TypeScript、火山引擎/外部模型 OpenAPI）建议保持一致；具体 UI 组件库、图表库、Agent 框架、开源模型可自由选择。 |
| **视频生成必须端到端跑通吗？** | 基础要求是端到端可运行（哪怕单条链路、模型可替换为轻量版本）；视频效果不是评审重点，更看重工程链路完整性与创新思路探索。 |
| **数据从哪里来？** | 分发与转化数据可使用 mock 数据或自建模拟服务，重点在于数据结构设计、归因逻辑与可视化呈现。 |
| **素材合规与版权如何处理？** | 可使用公开可商用素材或自有素材，重点是设计出"合规审核 + 真实性校验"的工作流。 |
| **可以借用 AI 编程吗？** | 完全可以，推荐使用 Trae 等 AI 编程工具，但需对架构和代码质量做好把控。 |
| **可以使用第三方库吗？** | 完全可以，开源社区的任何工具和方案都可以使用。 |
| **如何提交作业？** | 给定可访问域名或本地启动访问，并附上 README、架构说明与生成效果 demo（建议附 1–2 条端到端生成的视频作品）。 |
| **可以组队吗？** | 鼓励 2–3 人组队协作，最多不超过 3 人。建议分工明确。 |
