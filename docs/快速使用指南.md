# 快速使用指南

## 🎯 已完成的改进

### 问题解决
之前完成的A/B测试和多因子归因功能使用了独立的mock数据，没有与真实的视频生成记录关联。现在已修复：

✅ **新增数据库表**
- `video_factors` - 记录每个视频生成时使用的创作因子
- `video_publishing_records` - 记录视频发布和Mock效果数据

✅ **智能数据生成**
- 基于创作因子智能生成效果数据
- 不同因子组合有不同的效果权重
- 例如："痛点提问"开场比"直接展示"完播率高15%

✅ **数据关联**
- A/B测试结果现在会从数据库读取真实发布数据
- 归因分析会使用真实的视频因子数据
- 如果没有真实数据，会自动回退到示例数据

✅ **示例数据**
- 已初始化50个视频的完整数据
- 总播放量约105万
- 平均完播率93%
- 平均转化率15%
- 3个示例A/B测试实验

## 🚀 立即使用

### 1. 查看A/B测试
访问 `http://localhost:5173`（前端地址），导航到 **A/B测试** 页面，你会看到：
- 3个示例实验已创建
- 每个实验有对照组和实验组
- 展示了不同策略的转化率对比

### 2. 查看多因子归因分析
导航到 **多因子归因分析** 页面，你会看到：
- 50个视频的因子数据
- 各因子对效果的影响分析
- 优化建议

### 3. 生成新视频数据

#### 记录视频因子
```typescript
import { recordVideoFactors } from '../utils/api';

// 视频生成完成后
await recordVideoFactors(projectId, {
  openingStyle: '痛点提问',
  bgmStyle: '节奏感强',
  voiceoverStyle: '活泼热情',
  colorTone: '暖色调',
  duration: 15,
  sceneCount: 3,
  productName: '轻薄羽绒服'
});
```

#### 发布视频
```typescript
import { publishVideo } from '../utils/api';

// 发布视频并自动生成Mock数据
await publishVideo(projectId, {
  platform: 'douyin',
  productName: '轻薄羽绒服',
  experimentId: 'exp_xxx',  // 可选：关联A/B实验
  variantId: 'variant_a'     // 可选：指定变体
});
```

## 🔧 重新初始化数据

如果需要重新生成示例数据：

```bash
cd server
node scripts/initVideoData.js
```

这会：
1. 清空现有视频数据
2. 生成50个新的视频记录
3. 创建3个示例A/B测试实验
4. 显示数据统计

## 📁 新增文件

- `server/db/schema.sql` - 新增表结构
- `server/services/videoFactorService.js` - 视频因子服务
- `server/routes/videoFactors.js` - API路由
- `server/scripts/initVideoData.js` - 数据初始化脚本
- `frontend/src/utils/api.ts` - 前端API函数
- `docs/视频因子与AB测试数据集成说明.md` - 详细说明文档

## 🎓 学习要点

1. **数据结构设计** - 如何设计"因子→效果"的数据模型
2. **归因逻辑** - 如何分析哪个因子影响最大
3. **智能Mock** - 如何基于因子生成合理的效果数据
4. **前后端集成** - 如何让前后端数据流打通

## ⚠️ 注意事项

1. **数据持久化** - 所有数据存在 `server/data/app.db`
2. **外键约束** - 发布记录需要有效的项目ID
3. **实时更新** - 每次发布视频都会更新统计
4. **Mock数据** - 效果数据仅用于演示

## 💡 扩展建议

如果需要连接真实电商后台，可以：

1. 在 `publishVideo` 方法中添加真实的发布API调用
2. 在发布后调用电商平台的统计API获取真实效果数据
3. 将真实数据写入数据库替代Mock数据

这样就能实现"数据回流反哺"功能。
