## 二、各项目核心优点

### 2.1 Toonflow（⭐ 最值得借鉴）

**优点 1：三层 Agent 协作体系**
- 决策层（与用户交互）→ 执行层（具体任务）→ 监督层（质量审核）
- 决策层不执行任务，执行层不自审，监督层只提建议不改决策
- 每个 Agent 职责清晰，避免自我审查偏差

**优点 2：Skill 文件化配置**
- 所有 Agent 提示词外化为 Markdown 文件，支持在线编辑调优
- 11 种画风 Skill + 12 种故事类型 Skill + 制作技法 Skill
- 无需改代码即可调整 AI 行为

**优点 3：严格的资产引用体系**
- 角色/场景/道具统一管理，每个资产有唯一 ID
- 分镜中通过 ID 精确引用，监督层红线审核
- 衍生资产机制（角色换装、场景换时段等）

**优点 4：自动路由生成**
- 文件系统路由，`routes/` 下的文件自动映射为 API 端点
- 约 167 个 REST API，开发效率极高

---

### 2.2 ArcReel（⭐ Agent 编排最佳）

**优点 1：编排 Skill + 聚焦 Subagent**
- 主 Agent 极轻量，只做状态检测和任务分发
- 重上下文任务由 Subagent 独立完成，保护主 Agent context
- Subagent 不能 spawn subagent，多步工作流通过主 Agent 链式 dispatch

**优点 2：多供应商抽象层**
- Registry + Factory 模式统一 5 大供应商（Gemini/火山/Grok/OpenAI/Vidu）
- 图片/视频/文本三套后端协议清晰分离
- 支持自定义 OpenAI 兼容 API 接入

**优点 3：多层一致性保障**
- 角色设计图（四格设计稿）→ 分镜图参考 → 视频起始帧
- 场景设计图 + 道具三视图
- 风格参考图全局约束

**优点 4：生产级工程实践**
- bwrap 沙箱隔离、JWT 认证、异步任务队列
- 费用追踪、剪映草稿导出
- SSE 实时推送 + 会话管理

---

### 2.3 Story-Flicks（⭐ 端到端最完整）

**优点 1：完整可运行的视频生成管线**
- LLM 生成故事 → 图片生成 → TTS 配音 → 字幕渲染 → 视频合成
- 全链路打通，可直接产出带字幕的配音视频

**优点 2：免费 TTS + 字幕同步**
- Microsoft Edge TTS，400+ 语音，支持中英日韩
- WordBoundary 事件自动生成 SRT 字幕，与语音精确对齐
- 语速控制、多语言支持

**优点 3：MoviePy 视频合成**
- Ken Burns 效果（1.2x 缩放 + 水平平移）
- 白字黑边描边字幕，自动换行
- 多场景拼接，H.264 + AAC 编码

**优点 4：多 Provider 统一适配**
- OpenAI / 阿里云 / DeepSeek / Ollama / SiliconFlow
- 统一 OpenAI SDK 兼容接口，切换只需改配置

---

### 2.4 MovieStudio（⭐ Agent 架构参考）

**优点 1：CrewAI 多 Agent 编排**
- 6 个专业角色 Agent（编剧/视觉/角色/摄影/音效/导演）
- Sequential 顺序执行 + Context 上下文传递
- Agent 定义模式：role/goal/backstory

**优点 2：专业影视制作流程**
- 三幕结构、镜头类型、相机运动
- EDL 编辑决策列表
- 风格预设系统（电影/纪录片/黑色电影等）

import edge_tts
from edge_tts import SubMaker

async def generate_voice(text, voice_name="zh-CN-XiaoxiaoNeural"):
    communicate = edge_tts.Communicate(text, voice_name)
    submaker = SubMaker()
    
    audio_data = b""
    async for chunk in communicate.stream():
        if chunk["type"] == "audio":
            audio_data += chunk["data"]
        elif chunk["type"] == "WordBoundary":
            submaker.create_sub((chunk["offset"], chunk["duration"]), chunk["text"])
    
    return audio_data, submaker.generate_subs()  # 返回音频 + SRT字幕

#### 方案 3：简化版 Agent 编排（借鉴 Toonflow）

```javascript
// 三层 Agent 架构（简化版）
class AgentOrchestrator {
  constructor() {
    this.decisionAgent = new DecisionAgent();   // 决策层
    this.executionAgent = new ExecutionAgent(); // 执行层
    this.supervisionAgent = new SupervisionAgent(); // 监督层
  }
  
  async run(userInput) {
    // 1. 决策层分析需求
    const plan = await this.decisionAgent.analyze(userInput);
    
    // 2. 执行层生成剧本
    const script = await this.executionAgent.generateScript(plan);
    
    // 3. 监督层审核
    const review = await this.supervisionAgent.review(script);
    
    if (review.score >= 'B') {
      // 4. 执行层生成视频
      return await this.executionAgent.generateVideo(script);
    } else {
      // 根据建议修改后重试
      return this.run(review.suggestions);
    }
  }
}
```

## 四、推荐借鉴优先级

### 第一优先级（投入产出比最高）

1. **Edge TTS 配音** → 免费、0.5天、立即可用
2. **字幕渲染** → MoviePy、0.5天、视觉效果提升明显
3. **多分镜拼接** → FFmpeg、1天、从"单片段"到"完整视频"
4. **失败重试** → 异步队列、0.5天、稳定性提升

### 第二优先级（差异化亮点）

5. **简化版 Agent 编排** → 2天、比赛加分项
6. **BGM 混音** → FFmpeg、0.5天、视频质量提升

### 第三优先级（时间充裕再做）

7. **分镜编辑器** → 2天、UI 交互提升
8. **Skill 文件化** → 0.5天、可维护性提升

## 五、总结

| 项目 | 最值得借鉴的点 | 适合我们的场景 |
|------|--------------|--------------|
| **Toonflow** | 三层 Agent + Skill 文件化 + 资产引用 | 剧本生成质量、Agent 编排 |
| **ArcReel** | 多供应商抽象 + 视频拼接 + 转场 | 视频后处理、多模型支持 |
| **Story-Flicks** | Edge TTS + 字幕 + MoviePy 合成 | 配音、字幕、快速出片 |
| **MovieStudio** | CrewAI 编排 + 风格预设 | Agent 架构设计参考 |

**最佳策略**：以 Story-Flicks 的端到端执行能力为基础，融入 Toonflow 的 Agent 编排思想，用 ArcReel 的视频拼接方案做后期处理。
